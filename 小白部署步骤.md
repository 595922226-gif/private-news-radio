# 小白部署步骤

下面这份说明是给完全不懂代码的人看的。你先不用急着做完，后面可以按步骤一点点来。

## 第 1 步：准备 GitHub

你要准备：

- 一个 GitHub 账号

你要做：

1. 打开 GitHub。
2. 新建一个仓库，名字建议叫：`private-news-radio`。
3. 把本项目文件上传到这个仓库。

成功后你会看到：

- GitHub 页面里有 `config`、`scripts`、`docs`、`.github` 这些文件夹。

## 第 2 步：准备 OpenAI API Key

你要准备：

- OpenAI 账号
- 可用的 API Key

你要做：

1. 进入 OpenAI API 平台。
2. 创建一个新的 API Key。
3. 复制这个 Key。
4. 回到 GitHub 仓库。
5. 点 `Settings`。
6. 点 `Secrets and variables`。
7. 点 `Actions`。
8. 点 `New repository secret`。
9. 名字填：`OPENAI_API_KEY`。
10. 内容粘贴你的 API Key。
11. 点保存。

成功后你会看到：

- GitHub Secrets 里出现 `OPENAI_API_KEY`。
- 你看不到 Key 内容，这是正常的。

## 第 3 步：准备 Cloudflare R2

你要准备：

- Cloudflare 账号
- 一个 R2 存储桶
- 一个公开访问地址

你要做：

1. 打开 Cloudflare。
2. 进入 `R2 Object Storage`。
3. 创建一个 bucket，名字可以叫：`private-news-radio`。
4. 开启公开访问，或者绑定一个自定义域名。
5. 创建 R2 API Token。
6. 记下这些信息：
   - Account ID
   - Access Key ID
   - Secret Access Key
   - Bucket 名字
   - 公开访问地址，比如 `https://radio.example.com`

然后回到 GitHub Secrets，新增：

```text
R2_ACCOUNT_ID
R2_ACCESS_KEY_ID
R2_SECRET_ACCESS_KEY
R2_BUCKET
R2_PUBLIC_BASE_URL
```

成功后你会看到：

- GitHub Secrets 里出现上面 5 个名字。

## 第 4 步：准备邮箱发送

你要准备：

- 一个可以发送邮件的邮箱
- SMTP 服务信息

需要填到 GitHub Secrets：

```text
SMTP_HOST
SMTP_PORT
SMTP_USER
SMTP_PASSWORD
EMAIL_FROM
EMAIL_TO
```

举例：

```text
SMTP_PORT=587
EMAIL_FROM=你的发送邮箱
EMAIL_TO=你手机上接收的邮箱
```

注意：

- 很多邮箱的 `SMTP_PASSWORD` 不是登录密码，而是“授权码”。
- QQ 邮箱、163 邮箱、Outlook、Gmail 都有自己的 SMTP 设置方式。

成功后你会看到：

- GitHub Secrets 里出现 6 个邮箱相关名字。

## 第 5 步：手动测试一次

你要做：

1. 打开 GitHub 仓库。
2. 点上方的 `Actions`。
3. 左侧点 `Daily Private News Radio`。
4. 右侧点 `Run workflow`。
5. 再点绿色按钮确认运行。

成功后你会看到：

- 运行记录变成绿色对勾。
- 你的邮箱收到一封“全球情报早餐”。
- 邮件里有音频链接和文字版内容。

## 第 6 步：确认自动运行

这个项目已经设置为每天东京时间 7:30 自动运行。

你不需要每天打开电脑。
你不需要 iPhone 开 VPN。
你只需要早上打开邮箱，或者点快捷指令播放。
